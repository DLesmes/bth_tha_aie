# Implementation Approach

## Working Prototype / Proof-of-Concept (POC)

• Local Sandboxing: Built as a modular python application isolated via the uv package manager to ensure fast local dependency mapping without container overhead.

• Data Layering: Ingestion files are split into physical `data/public` and `data/private` directories to simulate distinct corporate access layers.

• Offline Verification: Includes a programmatic test suite (`test_graph.py`) that mocks user states (admin vs. collaborator). This verifies that the routing paths, FGA filtering boundaries, and fallback text strings function offline before turning on web services or UI dashboards.

## RAG Pipeline Execution Details

• Chunking Strategy: Documents are parsed using `PyPDFLoader` and broken down using a `RecursiveCharacterTextSplitter` configured for a 1000-character chunk size and 150-character overlap. During ingestion, critical security tags (`doc_id` and `allowed_role`) are explicitly injected into each chunk's metadata.

• Embedding Engine: Handled locally via Hugging Face `all-MiniLM-L6-v2` to process vectors quickly and eliminate network dependencies during indexing.

• Retrieval & Security Interception: A two-stage process. First, the Retriever Agent uses Gemini to optimize and rewrite the query terms for semantic search. Second, the chunks are intercepted by a LangChain `FGARetriever` wrapper, which screens resource access via Auth0 FGA. Unauthorized document pieces are dropped before reaching generation.

• Prompt Design: Built around strict factual grounding. The Specialist Agent is injected with a system instruction to answer using only the retrieved context block. If payloads are stripped by FGA, the node defaults to a safe refusal statement.

## Handling Ambiguous or Out-of-Scope Queries

• The Front Agent Guardrail: Every user prompt lands first on the Front Agent router node, preventing unnecessary database lookups or open-ended generation.

• Deterministic Keyword Filters: If a query falls completely outside the scope of hirebooth.com policies or documentation (evaluated via strict matching constraints), the system intercepts it immediately and routes execution straight to the END state with a hardcoded assistant template—bypassing the LLM, Weaviate, and Auth0 FGA to minimize token costs.

• Banter Bypass: Standard conversational remarks (e.g., greetings) bypass vector lookups entirely and route straight to the Specialist Agent for an immediate, friendly response.

## Evaluation Approach & Quality Measurement

• Runtime Verification (Inline Quality Control): The Reviewer Agent serves as an active, real-time evaluation step. It validates generated text against original text fragments to check for unverified inferences or hallucinations. If errors are caught, it triggers a self-healing iteration loop (capped at 3 retries).

• Offline Golden-Dataset Appraisals: CI/CD pipelines utilize RAGAS or LangSmith datasets to measure:

    • Faithfulness: Confirming that every statement can be traced back to source chunks.

    • Answer Relevance: Quantifying how directly the output solves the user prompt.

    • Context Precision: Verifying that the FGA-authorized chunk payload contains minimal noise.

## Key Risks and Open Questions

• FGA Batch Interception Latency: In production with heavy traffic, batch authorization for many chunks can create latency bottlenecks.

• Token Inflation via Iterative Retries: If Specialist and Reviewer agents enter recursive loops frequently, token consumption will spike.

• Privilege Sync Lag: Potential synchronization lag between enterprise user directories (e.g., Active Directory) and the centralized Auth0 FGA tuple-store when access rights change in real time.
